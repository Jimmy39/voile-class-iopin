To use, copy voile_interface_pin.h and depends libries, creat voile_conf.h for defining micro. Then include voile_interface_pin.h.

Enumeration Type Documentation

◆ voile_io_mode_t

enum voile_io_mode_t

IO mode selectors.

Enumerator
IOmodeInput 

As return value: The I/O is input mode, and both pull-up and pull-down resistors are disabled.
As input parameter: Set the I/O mode to input and disable pull-up and pull-down resistors.

IOmodeInputWithPullUpResistor 

As return value: The I/O is input mode, pull-up resistor is enabled and pull-down resistor is disabled.
As input parameter: Set the I/O mode to input, enable pull-up resistor and disable pull-down resistor.

IOmodeInputWithPullDownResistor 

As return value: The I/O is input mode, pull-down resistor is enabled and pull-up resistor is disabled.
As input parameter: Set the I/O mode to input, enable pull-down resistor and disable pull-up resistor.

IOmodeQuasiBidirectional 

As return value: The I/O is quasi-bidirectional mode.
As input parameter: Set the I/O mode to quasi-bidirectional mode.

IOmodePushPull 

As return value: The I/O is push-pull mode.
As input parameter: Set the I/O mode to push-pull mode.

IOmodeOpenDrain 

As return value: The I/O is open drain mode.
As input parameter: Set the I/O mode to open drain mode.

IOmodeHold 

As input value: Do nothing.

IOmodeError 

io_mode number should less than IOmodeError.

Initialise or reinitialise a IO, and enable it.

Parameters
[in]pin_p:[voile_const_pin_t *] The pointer of pin_p object.
Sample
pin_init(&myIo);
Details
Initialise or reinitialise a IO, and enable it.
If there is a configuration error in the devicelist.h, a error will be returned via voile_errorno
  • hardwareUnsupportedError means that you are using a feature not supported by the hardware. For example, if you are initializing an I/O expansion chip but using pins that are only in input mode to drive the I/O expansion chip, this error would be triggered.
  • configError means that there are invalid parameters in the configuration of devicelist.h.
Note
You should set mode and write value for output befor init.
In some hardware that does not support atomic operations, this function may initialize and enable the entire port.

Set mode for sigle io.

Parameters
[in]this:[voile_const_pin_t *] This pin object.
[in]mode:[voile_io_mode_t] Which mode select to use.

| Value | As input | As output | | — | — | — | | IOuninit | Behavior is undefined. If this I/O can be independently initialized and deinitialized, it will be deinitialized. | The I/O is uninitialized. | If your hardward unsupport the mode you slect, function will return hardware-unsupported-error and downgrade to most similar mode.
For details:

  • The input mode with pull-up or pull-down resistor will downgrade to input with nothing.
  • The quasi-bidirectional mode: will downgrade to open-drain then push-pull.
  • The open-drain mode will downgrade to quasi-bidirectional then push-pull.
  • The push-pull mode will downgrade to quasi-bidirectional then open-drain.
